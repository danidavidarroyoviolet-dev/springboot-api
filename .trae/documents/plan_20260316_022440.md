### 1. Diagnosis (Why 404?)
The issue is in the `RewritePath` configuration in `api-gateway/src/main/resources/application.yml`:
- **Config**: `RewritePath=/api/polizas/(?<segment>.*), /polizas/${segment}`
- **Request**: `GET /api/polizas`
- **Problem**: The regex `(?<segment>.*)` expects something *after* `/api/polizas/`. Since the request is exactly `/api/polizas` (no trailing slash), the regex doesn't match properly, or if it does, it might be rewriting it to just `/polizas` but the service expects `/polizas` to be the root of the controller.
- **Better Approach**: Use `StripPrefix=1`. This is the standard way to map `/api/service-name/**` -> `service-name/**`.
    - Request: `/api/polizas` -> Strip `/api` -> Forward `/polizas` (Matches Controller).
    - Request: `/api/polizas/1` -> Strip `/api` -> Forward `/polizas/1` (Matches Controller).
